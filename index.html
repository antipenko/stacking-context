Контекст наложения

это концепция трехмерного разположения HTML элементов вдоль оси Z , по отношению к пользователю, находящемуся перед экраном.

Элементы с общими родителями, перемещающиеся на передний или задний план вместе.

Контекст наложения -

Расположение внутри контекста

  1. Корневой элемент контекста, т.е. элемент, образовавший контекст наложения;
  2. Позиционированные элементы (+ потомки) с отрицательным z-index.
  3. Не позиционированные элементы располагаются по порядку;
  4. Позиционированные элементы (+ потомки) со значением z-index в auto;
  5. Позиционированные элементы (+ потомки) с положительным значением z-index.

Контекст наложения без z-index

  1. Фон и границы корневого элемента
  2. Не позиционированные блоки, в порядке появления в HTML
  3. Позиционированные блоки, в порядке появления в HTML

Контекст наложения без z-index

Контекст наложения c z-index

  1. Не позиционированные элементы в порядке появления на странице
  2. Позиционированные элементы в порядке увеличения z-index

Контекст наложения c z-index

Контекст наложения и float

  1. Фон и границы корневого элемента
  2. Не позиционированные блоки, в порядке появления в HTML
  3. Блоки с float, отличным от none
  4. Позиционированные блоки, в порядке появления в HTML

Контекст наложения и float

Формирование КН

  • является корневым элементом (HTML),
  • позиционирован абсолютно, фиксировано или относительно с z-index значением отличным от "auto"
  • flex элемент с z-index отличным от  "auto"
  • элементы с opacity меньше чем 1
  • элементы с transform отличным от "none"

Ключ к пониманию

На странице может быть несколько контекстов наложения. Взаимодействовать через z-index могут только дочерние элементы одного контекста.

Элемент А в контексте 1 и элемент B в контексте 2 не могут взаимодействовать через z-index. Таким образом если элементы А и В пересекаются, то их z-index ничего не решает, важен только z-index их родителей.
